# Software Requirements Specification

| ID | Parent | Priority | Requirement | Acceptance Criteria |
| --- | --- | --- | --- | --- |
| SW-001 | SYS-001 | must | The software shall implement: users must be able to record basic plant observations from a command-line interface. | Automated or manual tests can verify the behavior.<br>Verification evidence is recorded before release. |
| SW-002 | SYS-002 | should | The software shall implement: the app should store plant observation entries in a local file. | Automated or manual tests can verify the behavior.<br>Verification evidence is recorded before release. |
| SW-003 | SYS-003 | must | The software shall implement: users must be able to list previously recorded plant observations. | Automated or manual tests can verify the behavior.<br>Verification evidence is recorded before release. |
| SW-004 | SYS-004 | should | The software shall implement: the system should provide a traceable test plan for each plant observation requirement. | Automated or manual tests can verify the behavior.<br>Verification evidence is recorded before release. |
| SW-005 | SYS-005 | should | The software shall implement: final acceptance should require human review before release. | Automated or manual tests can verify the behavior.<br>Verification evidence is recorded before release. |
